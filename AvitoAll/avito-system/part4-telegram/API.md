# Part 4: Telegram Bot — Описание и API

## Назначение

Telegram-бот для получения уведомлений о найденных товарах и быстрого управления системой. Дублирует часть функционала веб-панели для мобильного использования.

## Функционал

1. **Уведомления** — автоматическая отправка сообщений о новых товарах с AI-вердиктами
2. **Быстрые команды** — добавление/удаление поисков, просмотр статуса
3. **Контроль доступа** — whitelist пользователей

---

## Что получает

### От Backend API

| Endpoint | Что получает | Когда |
|----------|-------------|-------|
| `GET /api/v1/items/new?since=...` | Новые товары | Каждые NOTIFY_INTERVAL сек |
| `GET /api/v1/searches` | Список поисков | По команде /list |
| `GET /api/v1/stats` | Статистика | По команде /status |

**Формат нового товара:**
```json
{
  "id": "7867391303",
  "title": "iPhone 12 Pro, 128 ГБ",
  "price": 15000,
  "location": "Москва",
  "url": "https://www.avito.ru/7867391303",
  "ai_verdict": "OK",
  "ai_score": 8,
  "ai_summary": "Хорошее состояние, без видимых дефектов",
  "ai_defects": [],
  "greeted": true,
  "found_at": "2026-02-03T14:30:00"
}
```

**Формат статистики:**
```json
{
  "searches_active": 5,
  "items_today": 12,
  "items_by_verdict": {"OK": 30, "RISK": 50, "SKIP": 65},
  "token_valid": true,
  "token_hours_left": 12.5
}
```

### От пользователя (Telegram)

| Команда | Формат | Пример |
|---------|--------|--------|
| `/start` | — | — |
| `/add` | `/add <query> [мин-макс] [доставка]` | `/add iPhone 12 Pro 10000-25000 доставка` |
| `/list` | — | — |
| `/remove` | `/remove <id>` | `/remove 3` |
| `/status` | — | — |
| `/stop` | — | — |

## Что отправляет

### На Backend API

| Endpoint | Что отправляет | Когда |
|----------|---------------|-------|
| `POST /api/v1/searches` | Новый поиск | По команде /add |
| `DELETE /api/v1/searches/{id}` | — | По команде /remove |
| `PUT /api/v1/searches/{id}` | `{enabled: false}` | По команде /stop (для всех) |

**Формат создания поиска:**
```json
{
  "query": "iPhone 12 Pro",
  "price_min": 10000,
  "price_max": 25000,
  "delivery": true,
  "location_id": 621540
}
```

### Пользователю (Telegram)

| Событие | Формат сообщения |
|---------|-----------------|
| Новый товар OK | ✅ + фото + детали |
| Новый товар RISK | ⚠️ + детали + предупреждение |
| /add успех | "Поиск добавлен: ..." |
| /list | Список поисков |
| /status | Статистика |

## Команды бота

### /start

Приветственное сообщение с помощью.

**Ответ:**
```
👋 Avito Monitor Bot

Команды:
/add <запрос> [мин-макс] [доставка] — добавить поиск
/list — список поисков
/remove <id> — удалить поиск
/status — статус системы
/stop — остановить все поиски
```

---

### /add <query> [мин-макс] [доставка]

Добавить новый поиск.

**Парсинг:**
```
/add iPhone 12 Pro 10000-25000 доставка
     ──────────── ──────────── ────────
     query        price range   delivery flag
```

- Последнее слово "доставка" → `delivery=true`
- Паттерн `\d+-\d+` → `price_min`, `price_max`
- Остальное → `query`

**Примеры:**
```
/add iPhone 12 Pro                    → query="iPhone 12 Pro", delivery=true
/add iPhone 12 Pro 10000-25000        → + price_min=10000, price_max=25000
/add Samsung S24 доставка             → query="Samsung S24", delivery=true
/add Xiaomi 14 15000-30000 доставка   → полный набор
```

**Ответ успех:**
```
✅ Поиск добавлен

📱 iPhone 12 Pro
💰 10 000 — 25 000 ₽
📦 С Авито Доставкой
🌍 Вся Россия

ID: 3
```

**Ответ ошибка:**
```
❌ Укажите название устройства

Пример: /add iPhone 12 Pro 10000-25000 доставка
```

## Уведомления

Фоновый цикл проверяет `GET /api/v1/items/new?since=...` каждые `NOTIFY_INTERVAL` секунд.

### Формат уведомления OK

```
✅ iPhone 12 Pro, 128 ГБ

💰 15 000 ₽
📍 Москва
📦 С доставкой

🤖 AI: 8/10
Хорошее состояние, без видимых дефектов

💬 Приветствие отправлено

🔗 https://www.avito.ru/7867391303
```

### Формат уведомления RISK

```
⚠️ iPhone 12 Pro, 64 ГБ

💰 6 000 ₽
📍 Саратов
📦 С доставкой

🤖 AI: 3/10
Подозрительно низкая цена

⛔ Дефекты:
• Цена ниже рыночной на 70%
• Мало фотографий

🔗 https://www.avito.ru/7867391304
```

### SKIP не отправляются

Товары с `ai_verdict=SKIP` не генерируют уведомления.

## Контроль доступа

**Whitelist из .env:**
```env
TELEGRAM_ALLOWED_USERS=123456789,987654321
```

- Пустой список = доступ всем
- Все команды проверяют `user.id in allowed_users`
- Неавторизованным: "⛔ Нет доступа"
