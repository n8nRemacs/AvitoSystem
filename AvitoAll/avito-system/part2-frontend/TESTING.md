# Part 2: Frontend (Web Panel) — План тестирования

## Инструменты

- **Vitest** — unit-тесты
- **@vue/test-utils** — тестирование Vue-компонентов
- **MSW (Mock Service Worker)** — мок Backend API
- **Playwright** или **Cypress** — E2E (опционально)

## Структура тестов

```
part2-frontend/
└── src/
    └── __tests__/
        ├── setup.js              # MSW setup, глобальные моки
        ├── api.test.js           # Тесты HTTP-клиента
        ├── SearchesView.test.js  # Страница поисков
        ├── RulesView.test.js     # Страница правил
        ├── ItemsView.test.js     # Страница результатов
        ├── DialogsView.test.js   # Страница диалогов
        ├── StatusView.test.js    # Страница статуса
        ├── SearchForm.test.js    # Компонент формы
        ├── ItemCard.test.js      # Компонент карточки
        └── StatusBadge.test.js   # Компонент бейджа
```

## MSW Моки

Мок Backend API для изолированного тестирования:

```javascript
// Фиксированные ответы
GET  /api/v1/searches  → [{id:1, query:"iPhone 12", price_min:10000, ...}]
POST /api/v1/searches  → {id:2, query:"Samsung S24", ...}
GET  /api/v1/rules     → [{id:1, text:"iCloud Lock", is_preset:true, ...}]
GET  /api/v1/items     → [{id:"123", title:"iPhone 12", ai_verdict:"OK", ...}]
GET  /api/v1/stats     → {searches_active:3, items_total:50, ...}
```

## Unit-тесты

### api.test.js — HTTP-клиент
| Тест | Ожидание |
|------|----------|
| `api.getSearches()` | Возвращает массив поисков |
| `api.createSearch({query: "test"})` | POST вызывается, возвращает объект |
| `api.deleteSearch(1)` | DELETE вызывается |
| Заголовок X-Api-Key присутствует | Во всех запросах |
| Обработка 500 ошибки | Бросает Error |
| Обработка 401 | Бросает Error с сообщением |

### SearchesView.test.js
| Тест | Ожидание |
|------|----------|
| Рендер пустого списка | "Нет активных поисков" |
| Рендер с данными | Таблица с N строками |
| Клик "Добавить" | Открывается форма |
| Переключатель enabled | PUT вызывается |
| Кнопка удаления | DELETE вызывается, строка исчезает |

### SearchForm.test.js
| Тест | Ожидание |
|------|----------|
| Пустая форма | Кнопка "Сохранить" disabled |
| Заполнить query | Кнопка активна |
| Ввести price_min > price_max | Ошибка валидации |
| Submit с валидными данными | emit('submit', data) |
| Чекбокс "доставка" по умолчанию вкл | checked=true |

### RulesView.test.js
| Тест | Ожидание |
|------|----------|
| Предустановленные правила без кнопки удаления | Кнопка отсутствует |
| Пользовательское правило с кнопкой удаления | Кнопка есть |
| Переключатель enabled | PUT вызывается |
| Добавление правила | POST, новая строка |

### ItemsView.test.js
| Тест | Ожидание |
|------|----------|
| Рендер карточек | N карточек |
| Фильтр по вердикту OK | Только OK-карточки |
| Фильтр по поиску | Только из выбранного поиска |
| Пагинация | Загрузка следующей страницы |

### ItemCard.test.js
| Тест | Ожидание |
|------|----------|
| Вердикт OK → зелёный бейдж | class содержит "green" |
| Вердикт RISK → жёлтый бейдж | class содержит "yellow" |
| Вердикт SKIP → красный бейдж | class содержит "red" |
| Фото отображается | img src из image_urls[0] |
| Ссылка на Avito | href корректный |
| Дефекты отображаются | Список дефектов виден |
| Пометка "Приветствие отправлено" | Если greeted=true |

### StatusBadge.test.js
| Тест | Ожидание |
|------|----------|
| `verdict="OK"` | Текст "OK", зелёный фон |
| `verdict="RISK"` | Текст "RISK", жёлтый фон |
| `verdict="SKIP"` | Текст "SKIP", красный фон |

### StatusView.test.js
| Тест | Ожидание |
|------|----------|
| Токен валиден | Зелёный индикатор |
| Токен истёк | Красный индикатор |
| Счётчики отображаются | Числа из stats |

### DialogsView.test.js
| Тест | Ожидание |
|------|----------|
| Рендер списка диалогов | N строк |
| Клик → раскрытие истории | Сообщения видны |
| Смена статуса | PUT вызывается |

## Критерии прохождения

| Критерий | Требование |
|----------|-----------|
| Все unit-тесты зелёные | 100% |
| Покрытие компонентов | ≥ 70% |
| Покрытие api.js | ≥ 90% |
| Нет console.error в тестах | 0 |
| Все компоненты рендерятся без ошибок | 100% |

## Метрики

| Метрика | Цель |
|---------|------|
| Время загрузки страницы (dev) | < 2 сек |
| Время сборки (npm run build) | < 30 сек |
| Размер бандла (gzip) | < 500 KB |
| Все формы валидируют ввод | 100% |
| Обработка ошибок API (500, 401, сеть) | Уведомление пользователю |

## Запуск

cd part2-frontend
