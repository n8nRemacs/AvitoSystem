## 1. Контекст продукта (для дизайнера)

«Avito Monitor» — персональный инструмент мониторинга Avito и ценовой разведки для одного пользователя (владельца домашнего сервера). Продукт делает две вещи:

1. **Охота за лотом** — поллит выдачу Avito по сохранённому URL поиска, новые объявления прогоняются через LLM-классификацию состояния (рабочий / iCloud-блок / разбит / на запчасти) и дальше через LLM-матч критериям пользователя. Подходящие лоты улетают в Telegram (и/или Max) с кнопками действий.
2. **Рыночная разведка** — на основе той же выдачи строится статистика рынка: clean-медиана (только working лоты), тренды, distribution состояний, исторические минимумы. Уведомления-инсайты («медиана упала на 8% за неделю», «historical low», «всплеск предложения») приходят в тот же мессенджер.

Ключевые архитектурные решения, влияющие на UI:
- **URL-based профили** (ADR-001): юзер копирует готовый URL поиска из веб-Avito. Не делаем динамические формы фильтров — одно текстовое поле + парсер показывает «бренд / модель / категория».
- **Двойная вилка** (ADR-008): у каждого профиля две независимые вилки. **Search-вилка** широкая — что грузим в БД для статистики. **Alert-вилка** узкая — что улетает в мессенджер. По умолчанию `search = alert ± 25%`. Это **главный визуальный концепт V1** — компонент `DualPriceRange` должен быть выразительным.
- **Двухступенчатый LLM** (ADR-010): дешёвый classify состояния на всех лотах, дорогой match только на лотах в alert-зоне с подходящим состоянием. В UI это значит: каждый лот всегда имеет `condition_class` (плашка), но детальный LLM-вердикт есть не у всех.
- **Market intelligence** (ADR-009): на странице профиля 4 ключевых виджета — линейный график медианы за 30 дней с пунктирной alert-вилкой, гистограмма распределения, donut-диаграмма condition_distribution, лента market-событий.

Подробности смотри в `DOCS/TZ_Avito_Monitor_V1.md` и `DOCS/DECISIONS.md` (если Claude Design их видит) — но для дизайна достаточно этого документа.

### 2.2. Карта 8 экранов и переходов

| # | Экран | URL | Назначение |
|---|---|---|---|
| 1 | Dashboard | `/` | Сводка — карточки активных профилей + лента последних уведомлений + KPI |
| 2 | Profile Create Form | `/search-profiles/new` | Самый новый UX: URL Avito → парсер → DualPriceRange → overlay → LLM-критерии |
| 3 | Profile List | `/search-profiles` | Портфель мониторингов с inline-метриками |
| 4 | Profile Stats | `/search-profiles/{id}/stats` | Главная визуальная фишка — 4 виджета рыночной разведки |
| 5 | Listings Feed | `/listings` | Лента карточек с фильтр-чипами по condition_class и alert/market |
| 6 | Price Intelligence Report | `/price-intelligence/{id}` | Отчёт ценовой разведки: вилка + топ-5 / -5 + рекомендация |
| 7 | Listing Detail | `/listings/{id}` | Карточка лота с описанием, фото, LLM-вердиктом |
| 8 | Messenger Preview | (нет URL) | Как выглядит уведомление в TG / Max |

Переходы:
- `Dashboard` → клик по profile-card → `Profile Stats`
- `Dashboard` → кнопка «+ Новый профиль» → `Profile Create Form`
- `Profile List` → клик профиль → `Profile Stats`
- `Profile Stats` → клик «Посмотреть лоты» → `Listings Feed` (предфильтрован)
- `Profile Stats` → клик «Применить рекомендованную вилку» → toast + редирект обратно
- `Listings Feed` → клик карточка → `Listing Detail`
- `Listings Feed` → фильтр-чип → HTMX-обновление списка in-place
- `Profile Create Form` → save → `Profile List`

## 7. Что НЕ делаем (важные ограничения)

1. **Нет login-экрана** в мокапах — это просто форма, обе стилистики на ней неинформативны
2. **Не копируем логотип Avito** дословно. Можно сделать собственный значок (например, треугольник + лупа) и подписать «Avito Monitor» собственным шрифтом
3. **Не делаем мобильную версию** в этом ТЗ — V1 десктопный (homelab-юзер). Sidebar и грид для desktop ≥1280px
4. **Не используем Material/Bootstrap** компоненты. Опираемся на Tailwind + DaisyUI (это видно в кодовой базе `avito-monitor/app/web/templates/`)
5. **Не выдумываем новые экраны** не из списка §2.2. Если что-то не помещается на 8 экранов — сожми

## 8. Инструкция к генерации (ДЛЯ Claude Design)

**Что я хочу получить на выходе:**

1. **16 интерактивных URL-прототипов** (8 экранов × 2 стиля), сгруппированных по стилям
2. Каждый стиль — отдельный самодостаточный кликабельный прототип с работающей навигацией по сайдбару
3. Все экраны заполнены данными из §3 — буква в букву (имена, цены, проценты)
4. На каждом экране показано основное состояние (filled). Empty/loading/error — короткими врезками или подписями «← так выглядит empty», без полноценных отдельных экранов под состояния
5. Mессенджер-превью (экран 8) делается как отдельная страница «Style Guide / Notifications» внутри каждого стиля — не часть основной навигации

**Формат именования артефактов:**
```
[Стиль] - [Номер]. [Экран]
например:
"Trader-Dark - 1. Dashboard"
"Trader-Dark - 2. Profile Create"
...
"Avito-Cosplay - 8. Messenger Preview"
```

**Размер canvas:** 1440×900 по умолчанию (desktop).

**Доп. указания:**
- Используй Tailwind CSS токены, синхронизируемые с DaisyUI (это наш реальный stack)
- Шрифты подключай через Google Fonts (Inter, JetBrains Mono для Trader)
- Изображения лотов — placeholder.com или unsplash, размер 80×80, цвета — синие/серые тона iPhone
- Иконки — Lucide или Heroicons (см. §5.8 / §6.8)
- Графики — стиль Chart.js (можно нарисовать SVG-аппроксимацию)

**После генерации:** дай ссылки на оба прототипа отдельно, чтобы можно было быстро переключаться между Trader и Avito-Cosplay. После сравнения один выбираем как winning style и итерируем уже только его.
