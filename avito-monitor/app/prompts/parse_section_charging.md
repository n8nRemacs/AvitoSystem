ВАЖНО — правила оценки state:

**"ok"** — продавец дал положительный сигнал по этому признаку. Сюда входит ЛЮБОЕ из:
  - прямое подтверждение: «работает», «в порядке», «целый», «без проблем»
  - **отрицательное описание дефекта** (negation): «без трещин», «нет царапин», «не битый», «без сколов», «корпус не вскрывался»
  - общее положительное состояние, очевидно покрывающее признак: «корпус в идеале», «экран как новый», «без дефектов», «состояние нового устройства»

**"defect"** — продавец явно описал проблему данного признака. Учитывай эвфемизмы и завуалированные формулировки («требует ремонта», «менялся», «на айклауде», «на запчасти», «не сбрасывается без аккаунта»).

**"unknown"** — признак ВООБЩЕ не упомянут (ни прямо, ни через negation, ни через общее состояние). ЭТО fallback — используй ТОЛЬКО когда невозможно ни ok, ни defect.

evidence = короткая цитата (≤80 симв) из описания; null если state=unknown.

Примеры распознавания (общие паттерны):
  «без трещин и царапин» — negation → state=ok для соответствующих признаков
  «работает идеально, без нареканий» — общий позитив → state=ok если признак упомянут
  «требует ремонта», «на запчасти» — эвфемизм → state=defect
  «на айклауде заблокирован», «привязан к Google» — explicit defect → state=defect
  признак вообще не упомянут → state=unknown
