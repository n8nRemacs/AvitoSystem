  ok — устройство чистое.
  unknown — нет упоминаний.

ВАЖНО — правила оценки state:

**"ok"** — продавец дал положительный сигнал по этому признаку. Сюда входит ЛЮБОЕ из:
  - прямое подтверждение: «работает», «в порядке», «целый», «без проблем», «iCloud чист»
  - **отрицательное описание дефекта** (negation): «без блокировок», «не привязан к», «iCloud сброшен», «без аккаунта владельца»
  - общее положительное состояние, очевидно покрывающее признак: «полностью рабочий», «состояние нового устройства»

**"defect"** — продавец явно описал проблему данного признака. ВАЖНО распознавать эвфемизмы и завуалированные формулировки:
  - «на айклауде», «на айклауд заблокирован», «на iCloud» → locks.icloud_linked=defect
  - «привязан к Apple ID владельца», «не сбрасывается без логина» → locks.icloud_linked=defect
  - «привязан к Google», «FRP-locked», «MDM-locked» → locks.frp_locked=defect
  - «привязан к Mi/Samsung/Huawei аккаунту» → locks.vendor_account=defect (НЕ Apple!)
  - «нужен пароль владельца», «забыт пароль» → locks.passcode_forgotten=defect

**"unknown"** — признак ВООБЩЕ не упомянут (ни прямо, ни через negation, ни через общее состояние). ЭТО fallback — используй ТОЛЬКО когда невозможно ни ok, ни defect.

evidence = короткая цитата (≤80 симв) из описания; null если state=unknown.

Примеры распознавания:
  «На айклауде заблокирован» / «на iCloud» → locks.icloud_linked=defect (evidence: «На айклауде заблокирован»)
  «iCloud чист, готов к продаже» → locks.icloud_linked=ok
  «Без блокировок, чистый» → все *_locked / *_linked / vendor_account → ok
  Просто «iPhone 12 Pro Max 256GB» без упоминания блокировок → unknown
