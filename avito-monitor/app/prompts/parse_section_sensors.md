Дополнительные указания по ключам:

- sensors.touch_id (биометрия отпечатка пальца):
  defect — продавец прямо упоминает неработающий Touch ID, «сенсор отпечатка не работает», «touchID сломан».
  ok — продавец прямо подтверждает работу Touch ID.
  unknown — Touch ID не упомянут, либо модель явно без Touch ID (iPhone X и новее с Face ID).

ВАЖНО — правила оценки state:

**"ok"** — продавец дал положительный сигнал по этому признаку. Сюда входит ЛЮБОЕ из:
  - прямое подтверждение: «работает», «в порядке», «без проблем»
  - **отрицательное описание дефекта** (negation): «без проблем с Wi-Fi», «SIM ловит», «динамик не хрипит»
  - общее положительное состояние, ОЧЕВИДНО покрывающее признак: «всё работает», «без нареканий», «полностью рабочий», «все датчики в порядке»

**"defect"** — продавец явно описал проблему данного признака:
  - «не работает Wi-Fi» / «глушит сеть» → sensors.wifi=defect
  - «Bluetooth не подключается» → sensors.bluetooth=defect
  - «SIM не видит сеть» / «один разъём не работает» → sensors.sim=defect
  - «Touch ID не работает» → sensors.touch_id=defect (только для моделей с Touch ID)

ВАЖНО про Touch ID и Face ID: iPhone X и новее имеют Face ID, БЕЗ Touch ID. Если описание упоминает «Неполадка Face ID» / «не работает Face ID» / «камера TrueDepth неисправна» — это **sensors.other=defect**, НЕ sensors.touch_id. Для таких моделей sensors.touch_id всегда unknown (если только продавец явно не пишет «Touch ID работает» — что для iPhone X+ некорректно).

**"unknown"** — признак ВООБЩЕ не упомянут (ни прямо, ни через negation, ни через общее состояние). ЭТО fallback — используй ТОЛЬКО когда невозможно ни ok, ни defect.

evidence = короткая цитата (≤80 симв) из описания; null если state=unknown.

Примеры распознавания:
  «Всё работает, без нареканий» → sensors.wifi/bluetooth/sim/other=ok (общий позитив)
  «SIM + eSIM работает» → sensors.sim=ok
  «Face ID неисправен» (для iPhone X+) → sensors.other=defect, sensors.touch_id=unknown
  Описание без упоминания датчиков → unknown
