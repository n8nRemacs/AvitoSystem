- operability.parts_only (продаётся только на запчасти / на разбор):
  Intent-based detection — определи намерение продавца, а не состояние устройства.
  defect — explicit «на запчасти», «на разбор», «нерабочий, продаю как комплект», «только на детали», «разборка».
  ok — продавец продаёт как рабочее устройство (нет explicit указания «на запчасти»).
  unknown — нет явного указания ни в ту, ни в другую сторону.

ВАЖНО — правила оценки state:

**"ok"** — продавец дал положительный сигнал по этому признаку. Сюда входит ЛЮБОЕ из:
  - прямое подтверждение: «работает», «включается», «без проблем»
  - **отрицательное описание дефекта** (negation): «нет циклов перезагрузки», «не зависает», «без bootloop»
  - общее положительное состояние, ОЧЕВИДНО покрывающее признак: «полностью рабочий», «всё работает идеально», «без нареканий», «состояние нового устройства»

**"defect"** — продавец явно описал проблему данного признака. Учитывай эвфемизмы:
  - «не включается», «не загружается дальше яблока» → operability.no_boot / apple_loop=defect
  - «уходит в перезагрузку», «boot loop» → operability.boot_loop=defect
  - «на запчасти», «на разбор», «нерабочий», «как комплект на детали» → operability.parts_only=defect

**"unknown"** — признак ВООБЩЕ не упомянут (ни прямо, ни через negation, ни через общее состояние). ЭТО fallback — используй ТОЛЬКО когда невозможно ни ok, ни defect.

evidence = короткая цитата (≤80 симв) из описания; null если state=unknown.

Примеры распознавания:
  «Полностью рабочий, всё работает» → operability.no_boot/apple_loop/boot_loop=ok, parts_only=ok
  «На запчасти» / «на разбор» / «как комплект на детали» → parts_only=defect
  «Не включается», «не загружается дальше яблока» → no_boot=defect
  Описание без упоминания работоспособности → unknown
